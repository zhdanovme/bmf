# Правила работы с BMF-проектом

## Консистентность файлов

При любом изменении в этой папке — обновляй **все** затронутые файлы, чтобы они оставались согласованными между собой:
- YAML-сущности
- `_epics.yaml`, `_test-cases.yaml`
- `mock-frontend/` — если папка присутствует, обновляй мок-фронтенд параллельно с YAML

Изменения в разных файлах делай **параллельно** (несколько tool calls в одном сообщении).

## Валидация после правок

После внесения изменений в YAML — **обязательно** прогони валидацию:

```bash
npx ts-node utils/validate-schema.ts bmfs/{project}      # Валидация схемы
npx ts-node utils/check-references.ts bmfs/{project}     # Проверка ссылок
npx ts-node utils/validate-tcs-epics.ts bmfs/{project}   # Проверка epic:* ↔ tc:* соответствия
```
